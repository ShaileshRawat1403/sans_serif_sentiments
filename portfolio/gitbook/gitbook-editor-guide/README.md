# ✍️ GitBook Editor Guide

Use this guide to write, format, and structure content using GitBook's built-in editor.  
Whether you're drafting internal documentation or publishing public-facing help guides, the GitBook editor provides a flexible, accessible authoring experience.

---

## 🧭 Overview

The GitBook editor supports two writing modes:

- **Block mode** – A visual editor that lets you insert and rearrange content using blocks  
- **Markdown mode** – A code-style writing mode for users familiar with markdown syntax

You can switch between these modes anytime from the page settings.

---

## 🧱 Use the Block Editor

To start writing:

- Open any page in your GitBook space  
- Click inside the body area to activate the editor  
- Type `/` to open the command palette

Common content blocks:

- **Text** – Add standard paragraphs  
- **Heading** – Create H1, H2, H3 sections  
- **List** – Add bullet or numbered lists  
- **Quote** – Format tips, warnings, or citations  
- **Image** – Upload visuals directly  
- **Embed** – Add rich embeds (e.g., Figma, YouTube, Loom)  

> 🖼️  
> ![Block command menu with formatting options](images/gitbook-block-command-menu.png)  
> *Use `/` to access blocks such as text, headings, dividers, or images.*

To move a block, hover over the left edge and drag it using the handle.

---

## 📝 Use the Markdown Editor

If you prefer writing with syntax, switch to **Markdown mode**:

- Click the **page settings (•••)** icon in the upper right  
- Select **Switch to Markdown mode**  

Markdown supports:

| Task | Markdown Syntax |
|------|------------------|
| Heading 1 | `# Title` |
| Bold | `**bold**` |
| Italic | `_italic_` |
| Bullet list | `- Item` |
| Link | `[GitBook](https://gitbook.com)` |
| Image | `![Alt text](./image.png)` |
| Code block | ```` ```language ```` |

> 🖼️  
> ![Markdown syntax examples in GitBook](images/gitbook-markdown-editor-sample.png)  
> *Write using Markdown if you prefer minimal formatting and keyboard shortcuts.*

> 📘 Tip: Markdown is useful when syncing docs from GitHub repos or editing offline.

---

## 🗂️ Structure and Organize Pages

- Use the **sidebar** to manage hierarchy  
- Click **“+ New page”** to create a new doc  
- Drag pages to reorder or nest under sections  
- Rename pages using the context menu (•••)  
- Use folders to group related topics

> 🖼️  
> ![Sidebar with nested documentation pages](images/gitbook-sidebar-page-structure.png)  
> *Drag and drop pages in the sidebar to create structured, navigable documentation.*

---

## 🛠️ Common Editor Actions

| Action | How to Do It |
|--------|--------------|
| Add heading | Type `/heading` or use Markdown `#` |
| Insert image | Type `/image` or drag and drop |
| Add embed | Type `/embed` and paste the link |
| Publish page | Click **Publish** (top right) |
| Undo change | `Ctrl+Z` or `Cmd+Z` |
| Revert to draft | Click **Unpublish** under settings |

---

## 🔁 Switch Between Modes

- Open page settings (**•••**)  
- Select **Switch to Markdown** or **Switch to Blocks**  
- Preview formatting before publishing  
- Be aware: switching modes may reformat some content

> 🖼️  
> ![Toggle between Markdown and Block editor](images/gitbook-switch-editor-mode.png)  
> *Switch editing modes from the top-right page settings menu.*

---

## ✅ Outcome

After completing this guide, you should be able to:

- Write content using both block and markdown editors  
- Use formatting tools, embeds, and shortcuts  
- Organize pages using the sidebar  
- Switch between editing modes as needed
