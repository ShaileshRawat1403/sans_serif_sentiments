# 📚 GitBook Glossary

This glossary defines core GitBook terms and concepts. Each entry includes a description and links to the related guide.

---

| **Term** | **Definition** | **Related Guide** |
|----------|----------------|-------------------|
| **Access Control** | Permissions that define who can view or edit a workspace or space. | [Setup Guide](gitbook-setup-guide.md) |
| **Admin (Role)** | Full-access user who manages users, permissions, and space visibility. | [Setup Guide](gitbook-setup-guide.md) |
| **Block Mode** | Visual editing mode using content blocks (e.g. headings, lists, embeds). | [Editor Guide](gitbook-editor-guide.md) |
| **Draft** | Unpublished page or space, only visible to workspace members. | [Getting Started Guide](gitbook-getting-started.md) |
| **Editor (Role)** | Can create, edit, and publish pages but not manage users. | [Setup Guide](gitbook-setup-guide.md) |
| **Git Sync** | Syncs GitBook spaces with GitHub repositories. | [Overview](gitbook-overview.md) |
| **Markdown Mode** | Plain text editing with Markdown syntax. | [Writing Modes Reference](gitbook-writing-modes-reference.md) |
| **Page** | A single document within a space. | [Getting Started Guide](gitbook-getting-started.md) |
| **Publish** | Makes a page or space visible to external readers. | [Getting Started Guide](gitbook-getting-started.md) |
| **Role** | Defines permissions within a workspace (Admin, Editor, Viewer). | [Setup Guide](gitbook-setup-guide.md) |
| **Slash Command (`/`)** | Shortcut to insert content blocks in the editor. | [Editor Guide](gitbook-editor-guide.md) |
| **Space** | A group of pages organized around a specific topic. | [Overview](gitbook-overview.md) |
| **Viewer (Role)** | Read-only access to published or shared spaces. | [Setup Guide](gitbook-setup-guide.md) |
| **Workspace** | The top-level environment containing members and spaces. | [Overview](gitbook-overview.md) |

---

## 🧭 Tip  
Use **Ctrl+F / Cmd+F** to search. Need examples? See the [Editor Guide](gitbook-editor-guide.md).
