# 📚 GitBook Glossary

This glossary explains key terms used in GitBook, so you can navigate the platform with clarity and confidence.

All definitions are written for new users and non-technical contributors, with direct links to related guides where applicable.

---

## A

### **Access Control**  
The ability to define who can view, edit, or administer a space or workspace.  
> Related: [Setup Guide →](./gitbook-setup-guide.md)

---

## B

### **Block Mode**  
A visual writing mode where you add content using drag-and-drop blocks.  
Examples: text, headings, images, quotes, embeds.  
> Related: [Editor Guide →](./gitbook-editor-guide.md)

---

## D

### **Draft**  
A page or space that is not yet published or visible to public viewers.  
You can continue editing drafts privately until ready to share.

---

## E

### **Editor (Role)**  
A workspace member who can create, modify, and publish documentation but cannot manage workspace settings.  
> Related: [Setup Guide →](./gitbook-setup-guide.md)

---

## G

### **Git Sync**  
Optional feature allowing you to connect a GitHub repository to a GitBook space.  
Changes to Markdown files in the repo sync with GitBook automatically.

---

## M

### **Markdown Mode**  
A writing mode where you use plain-text formatting (e.g., `**bold**`, `# Heading`) instead of visual blocks.  
> Related: [Writing Modes Reference →](./gitbook-writing-modes-reference.md)

---

## P

### **Page**  
A single unit of content within a space.  
Pages can be nested under headings and grouped into folders for structure.

### **Publish**  
To make a draft page or space visible based on selected permissions (e.g., public or team-only).  
Publishing begins the trial period for GitBook if it's the first public space.

---

## R

### **Role**  
Permissions assigned to each user in a workspace.  
Default roles:  
- **Admin** – Full access to settings and members  
- **Editor** – Can write and publish  
- **Viewer** – Can read only

---

## S

### **Space**  
A collection of related pages grouped together as a documentation project.  
Each workspace can have multiple spaces.

### **Slash Command (`/`)**  
A shortcut used in the block editor to insert elements (e.g., `/heading`, `/quote`, `/image`).

---

## V

### **Viewer (Role)**  
A member with read-only access to private documentation. Cannot edit or comment.

---

## W

### **Workspace**  
Your main GitBook environment. A workspace can contain multiple documentation spaces and members with various roles.  
> Related: [Overview →](./gitbook-overview.md)

---

## 📌 Tips for Using This Glossary

- Use **Cmd/Ctrl + F** to search a term  
- Refer to the [Editor Guide](./gitbook-editor-guide.md) for usage examples  
- Terms may differ slightly depending on whether you're on a personal or team plan
