#  Writing Modes Reference: Markdown vs Blocks


GitBook offers two writing modes: **Markdown** and **Block editor**.  
This guide helps you understand the differences so you can choose the best editing experience for your workflow.

---

##  Overview


| Mode | Description |
|------|-------------|
| **Markdown** | Write using plain text and formatting symbols (e.g., `**bold**`) |
| **Block editor** | Use a visual interface with drag-and-drop blocks and slash commands |

You can switch between modes at any time from the **page settings menu (•••)**.  
Each mode supports core documentation needs, but the authoring experience varies.

---

## 🆚 Feature Comparison Table


| Feature | Markdown Mode | Block Mode |
|---------|----------------|------------|
| **Writing style** | Plain text syntax | Visual content blocks |
| **Formatting** | Markdown characters (e.g., `#`, `**`) | Rich text UI + `/` menu |
| **Embeds** | Supported via markdown or HTML | Supported via embed block |
| **Images** | Markdown syntax (`![Alt](link)`) | Drag-and-drop or `/image` |
| **Tables** | Requires markdown formatting | Native table block |
| **Real-time preview** | Yes | Yes |
| **Live collaboration** | Yes | Yes |
| **Best for** | Developers, technical writers | Designers, teams, non-coders |

##  When to Use Markdown Mode


Choose Markdown when you want to:

- Write quickly using keyboard-only input  
- Sync with GitHub or version control  
- Migrate existing Markdown docs  
- Control structure through minimal formatting

> 📘 Tip: Markdown is ideal for technical writers and developers working in docs-as-code environments.

---

##  When to Use Block Editor


Use Block mode if you prefer:

- A visual writing experience  
- Easy embedding (Loom, Figma, Google Docs, etc.)  
- Reordering sections using drag-and-drop  
- Collaborative, real-time content creation

> 📘 Tip: Block mode is more intuitive for non-technical contributors and teams managing knowledge bases.

---

##  Switching Between Modes


You can switch modes anytime:

- Open the **page settings (•••)**  
- Select **Switch to Markdown** or **Switch to Block editor**  
- Confirm the formatting preview

> ⚠️ Note: Converting complex layouts between modes may affect formatting. Review your content before publishing.
