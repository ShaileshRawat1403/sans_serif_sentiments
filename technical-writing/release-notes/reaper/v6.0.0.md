These release notes document the major enhancements introduced in **REAPER version 6.0**, a pivotal upgrade that significantly improved the visual workflow, routing capabilities, and customization for users.

## Track Control Panel Embedding

REAPER 6.0 introduced **embedded FX and send controls** directly within the Track Control Panel (TCP).

**What Changed:**

- You can now **embed plugin interfaces** directly into tracks in both the TCP and Mixer Control Panel (MCP).
- **Send slots** can also appear directly on the track interface, allowing faster signal routing adjustments.

**Why It Matters:**

This eliminates the need to open floating plugin windows for common tweaks. It's a major time-saver for those working with multiple inserts or complex routing setups, particularly in live performance or broadcast scenarios.

---

## Routing and Send Enhancements

Signal routing got a massive usability upgrade:

- New **per-track send slots** with drag-and-drop behavior
- Visual send paths are now more **intuitive**, with popovers and hover previews
- Quick-access **volume and pan controls for sends** right in the TCP

**Practical Benefit:**  
This lets users manage complex routing structures—like drum buses, submixes, and parallel chains—without constant window switching.

---

## Track Wiring View

One of the most visually impactful upgrades in REAPER 6.0 is the **Track Wiring Diagram**.

**What It Is:**

- A **graphical map** of your track-to-track signal flow
- Shows **sends, receives, and hardware outputs** visually
- Lets you quickly **trace and reroute audio** paths

**Why It Matters:**

Previously, complex routing often required diving into menus or plugin chains. With the wiring view, engineers can identify feedback loops, fix routing errors, and experiment with creative configurations—all in a single, interactive space.

## Extended Video Support

While REAPER is not primarily a video editor, v6.0 took a bold step forward in **multimedia compatibility**:

- Added support for **multiple video streams**
- Improved **frame-accurate scrubbing**
- More reliable performance on **macOS and Windows** systems with high-res video formats

**Use Case:**  
For sound designers, composers, and YouTubers, REAPER now functions as a viable sync tool for video scoring and dialogue replacement.

---

## New Theme Engine and Layout Tweaks

REAPER 6.0 shipped with a brand new **Default_6.0 Theme**—a cleaner, more scalable, and accessible UI.

**New Features:**

- **HiDPI-aware rendering** for high-resolution displays
- Dynamic sizing and layout flexibility based on screen real estate
- Customizable **layout scripts** to adjust elements based on user preferences

**Developer Note:**  
Themes can now include **WALTER scripts** for more nuanced layout control, giving visual designers full command over track behavior and component visibility.

---

## FX and Plugin Chain Improvements

Version 6.0 improved how FX chains behave:

- Drag-and-drop FX between tracks without glitches
- Pin frequently used FX windows to reduce clutter
- FX window docking now remembers last position and size

**Additional Benefit:**  
Routing inside the FX chain now mirrors track-level sends, enabling **parallel and split processing chains** in a visually intuitive format.

---

## Performance and Engine Refinements

Though most visual changes grabbed the headlines, several **under-the-hood optimizations** were introduced:

- Faster project load times (especially with video or large sample libraries)
- More responsive MIDI editing, especially under high CPU loads
- Better multicore CPU distribution, especially on Apple Silicon and Ryzen-based systems

---

## Compatibility Notes

REAPER 6.0 was built to be backward-compatible with REAPER 5.x projects. However:

- Some layout elements may appear differently (e.g., embedded sends)
- Scripts written for pre-6 themes may require tweaks
- Hardware routing templates from older setups might not reflect new TCP send views

It’s highly recommended to **save backups** before opening older projects.

---

## Should You Upgrade?

**Yes—if any of these apply to you:**

- You rely on heavy routing, parallel FX, or stem mixing
- You’re on a high-resolution display and struggle with UI scaling
- You edit audio-to-video and need better frame sync tools
- You prefer workflows with minimal window toggling

This is **not just a visual upgrade**—it’s a workflow rethinking that benefits both technical engineers and creative producers alike.

## Glossary

| Term                     | Definition                                                                 |
|--------------------------|----------------------------------------------------------------------------|
| **DAW**                  | Digital Audio Workstation – software used to record, edit, and produce audio |
| **TCP**                  | Track Control Panel – the vertical view of tracks on the left of REAPER    |
| **MCP**                  | Mixer Control Panel – the horizontal mixer view, typically shown at bottom |
| **FX Chain**             | A sequence of plugins applied to a track                                   |
| **Send Slot**            | A routing path from one track to another                                   |
| **Track Wiring View**    | A visual diagram showing all track connections and signal flows            |
| **WALTER**               | A scripting language used to customize themes in REAPER                   |
| **HiDPI**                | High Dots Per Inch – modern display resolution standard                    |
| **Parallel Processing**  | Applying FX simultaneously on separate signal paths for creative control   |
| **Frame Sync**           | Precise alignment of audio to specific video frames                        |
