# REAPER v6.0.0 Release Notes

REAPER 6.0 marked a pivotal moment in Cockos' development cycle—introducing a modernized user interface, a new default theme, and several under-the-hood improvements that significantly elevated both the user experience and technical depth.

This document serves as a **single source of truth** for understanding what changed in v6.0.0, why it matters, and how to use it effectively—especially if you're coming from version 5.x or new to the REAPER ecosystem.

---

## Who Should Read This

This guide is designed for:

- **REAPER 5.x users** upgrading to v6 and wanting a breakdown of interface and workflow changes.
- **Beginner musicians, engineers, and producers** exploring REAPER for the first time.
- **Intermediate DAW users** switching from other software like Pro Tools, Logic Pro, or Ableton and curious about what REAPER offers.

> If you’re unfamiliar with what a **DAW (Digital Audio Workstation)** is, stay tuned for our [What is a DAW?](../../conceptual-docs/what-is-a-daw.md) primer.

## New Default Theme

REAPER v6.0 introduced a modern **default theme**—more than just cosmetic.

- **Flat design elements** replaced the 3D look of REAPER 5.x, improving readability.
- **Dynamic layout adaptation** based on track width/height.
- **Smarter context switching**: The layout subtly adapts based on what you’re editing—whether that’s audio, MIDI, or automation.

> **Why it matters:** The UI now gets out of the way, letting you focus on your session. You don’t need to tweak dozens of visual settings just to see what matters.

---

## Track Control Panel (TCP) Redesign

The TCP was redesigned to support **responsive layouts**.

- TCP elements now **scale and reorganize** based on track height and screen space.
- You can configure **multiple layouts per track** (e.g., for folders, child tracks, busses).

> You can apply different layouts via `Options → Layouts → Track Panel`.

---

## Routing and Mixer Enhancements

- **Improved routing matrix UI**: More intuitive drag-and-drop patching, clearer visualization.
- **FX signal flow visualization**: See exactly where your plugins sit in the chain.
- **Send/receive displays** have been streamlined in the mixer and TCP.

> For advanced users: matrix-style routing in REAPER remains one of its most powerful features, now made easier for new users.

---

## Track Lanes and Item Grouping Updates

- New **track lane comping improvements** laid the foundation for v7.0’s full lane-based comping.
- **Grouped items** behave more predictably during slip editing and duplication.
- Item lanes maintain proper alignment across grouped tracks, even during timebase changes.

> If you record multi-mic drums or layered vocals, these changes reduce accidental misalignment.

---

## Envelopes & Automation Changes

- **Automation items** are now more stable and easier to move or copy between tracks.
- Envelopes now snap better to grid lines and item edges.
- **New preference**: Show/hide envelope lanes by default.

> REAPER’s automation engine stays light—but v6 improved visibility, snap precision, and overall stability.

---

## MIDI Editor Improvements

- You can now view and edit **multiple MIDI items across tracks** simultaneously.
- MIDI editor now supports **coloring by source track**, improving clarity in layered compositions.
- Improved **note snapping**, **quantization presets**, and **inline editing tools**.

> Users composing orchestral or electronic music will find these changes reduce repetitive clicks.

---

## FX Parameter Embedding

- You can now embed FX controls **directly into the TCP or MCP** (Mixer Control Panel).
- Drag plugin parameters into the TCP for quick access—no need to open the FX window every time.

> This greatly speeds up workflows when using virtual instruments or chain-heavy setups.

---

## Project Media Management

- REAPER v6 added a **Project Bay for FX chains and media items**.
- You can now manage large sessions with centralized access to:
  - Active and inactive media
  - FX chains and instances
  - Item states and routing

> Think of this as a consolidated media + plugin manager for power users.

---

## Performance and Under-the-Hood Tweaks

- Improved rendering engine for **smoother zooming and scrolling** on large sessions.
- Faster session loading and **reduced crash scenarios** when using unstable third-party plugins.
- Enhanced compatibility with:
  - macOS Mojave and Catalina
  - Windows 10 and 11
  - Linux via WINE (for adventurous users)

> In REAPER tradition, performance upgrades remain subtle but impactful. v6 continues to work well even on older machines.

## Glossary

| Term                     | Definition                                                                 |
|--------------------------|----------------------------------------------------------------------------|
| **DAW**                  | Digital Audio Workstation: software for recording, editing, mixing audio  |
| **TCP**                  | Track Control Panel: controls visible next to each track on the left      |
| **MCP**                  | Mixer Control Panel: a console-style view of tracks and buses             |
| **Comping**              | Selecting and blending takes into a final version                         |
| **Envelope**             | A graphical automation line that controls parameters over time            |
| **Track Lane**           | A visual layer for multiple takes or versions within a track              |
| **FX Parameter Embedding** | Displaying plugin controls directly on the track/mixer panel             |
| **Project Bay**          | A REAPER utility for managing media files, FX, and states in one place    |
| **Automation Item**      | A reusable envelope segment that can be copied and edited like media      |
| **WINE**                 | A compatibility layer that allows Windows software to run on Linux        |

## Summary

REAPER 6.0 introduced not just visual upgrades—but foundational improvements that set the tone for future innovation in versions 7.0 and beyond.

If you're coming from REAPER 5.x, this is the version where REAPER becomes **easier to read**, **faster to edit**, and **more capable in complex projects**.

> For users looking to upgrade without breaking habits—v6.0 is the perfect midpoint between the old and the new. You get speed, clarity, and control—without sacrificing your workflow muscle memory.
